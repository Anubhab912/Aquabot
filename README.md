# 🌊 AquaBot

AquaBot is a modern, dark-themed web application built with **FastAPI** and **HuggingFace** that delivers AI-powered weather intelligence for any location on Earth — from major cities to remote villages.

With a smart geocoding engine that resolves even the most obscure rural locations, AquaBot combines real-time Open-Meteo weather data with **Qwen 2.5-32B** AI insights to give you actionable farming advice and health tips tailored to current conditions. It includes a cinematic splash screen, an interactive Leaflet.js world map, 7-day forecasts, and a graceful rule-based fallback so it works with or without an API key.

## 🌟 Key Features

- **🗺️ Interactive World Map**: A live Leaflet.js map centers on your searched location as the background of the entire UI.
- **🔍 Global Location Search**: Resolves cities, villages, and rural areas using Open-Meteo geocoding with an OpenStreetMap/Nominatim fallback — no location too small.
- **🌡️ Real-Time Weather**: Current temperature, feels-like, humidity, wind speed, UV index, and precipitation from Open-Meteo.
- **📅 7-Day Forecast**: Scrollable daily forecast strip with weather icons, min/max temps, and rainfall per day.
- **🤖 AI Insights**: Weather summaries, farming tips, and health tips generated by Qwen 2.5-32B via the HuggingFace Inference API.
- **⚠️ Smart Alerts**: Automatic delta-score alerts that detect significant upcoming weather changes and warn you in advance.
- **🌱 Rule-Based Fallback**: A fully offline, rule-based engine kicks in gracefully when no HuggingFace token is provided.
- **🐳 Docker + Cloud Run Ready**: Production Dockerfile included, designed for Google Cloud Run deployment.

## 🚀 Setup & Installation

### 1. Prerequisites
- **Python 3.11+**
- A free [HuggingFace account](https://huggingface.co/settings/tokens) *(optional — app works without it using the fallback engine)*

### 2. Clone the Repository
```bash
git clone https://github.com/Anubhab912/Aquabot.git
cd Aquabot
```

### 3. Create a Virtual Environment
```bash
python -m venv venv

# On Windows:
.\venv\Scripts\activate
# On Mac/Linux:
source venv/bin/activate
```

### 4. Install Dependencies
```bash
pip install -r requirements.txt
```

### 5. Configure Environment Variables
1. Copy the example file: `copy .env.example .env`
2. Open `.env` and fill in your HuggingFace token:
```env
HF_API_TOKEN=hf_your_token_here
```
> **Note:** If `HF_API_TOKEN` is left empty, AquaBot will automatically use its built-in rule-based engine — no token required to run the app!

### 6. Run the App
```bash
uvicorn main:app --reload --port 8000
```
Open your browser at **http://localhost:8000**

## 🐳 Docker & Cloud Run

### Build & Run with Docker
```bash
docker build -t aquabot .
docker run -p 8080:8080 --env-file .env aquabot
```

### Deploy to Google Cloud Run
```bash
gcloud run deploy aquabot \
  --source . \
  --region asia-south1 \
  --allow-unauthenticated \
  --set-env-vars HF_API_TOKEN=your_token_here
```

## 📡 API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Serves the frontend UI |
| `GET` | `/health` | Health check — returns `{"status": "ok"}` |
| `GET` | `/geocode?city={name}` | Resolves a location name to lat/lon |
| `GET` | `/weather?city=&lat=&lon=` | Fetches weather + AI insights |

## 📁 Project Structure

```
Aquabot/
├── main.py             # FastAPI app, routes, geocoding logic
├── weather.py          # Open-Meteo weather fetching & parsing
├── ai_engine.py        # HuggingFace AI insight generation + fallback

├── models.py           # Pydantic data models
├── wmo_codes.py        # WMO weather code → description mapping

├── Dockerfile          # Production Docker image
├── .env.example        # Environment variable template

└── static/
    ├── index.html      # Frontend UI
    ├── style.css       # Glassmorphism dark theme styles
    └── app.js          # Frontend logic (geocoding, rendering)
```
